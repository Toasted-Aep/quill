## Hard constraints that shape every "how" below

1. **No custom HLSL / PixelShaderEffect.** Win2D built-ins only: GaussianBlur, Composite, Blend (26 modes), ColorMatrix, DisplacementMap, DistantSpecular/DistantDiffuse/PointSpecular, Morphology, Shadow, Transform2D, Transform3D, AlphaMask, Crop, Turbulence, LuminanceToAlpha, EdgeDetection, TableTransfer, HueRotation, Saturation, TemperatureAndTint, Scale, Opacity, ArithmeticComposite. Height-map lighting via DistantSpecularEffect is already proven here.
2. **WinUI 3 caches GradientStop mutations.** Changing a stop's Color/Offset on a live brush does not repaint. Swap the whole Brush object, or draw the thing in Win2D. Every colour-wheel / ring / gauge below obeys this.
3. **library.json is already 53 MB.** Raster and binary data live outside it, under `%LOCALAPPDATA%\Quill\...`, written temp + flush + atomic-replace. In-place writes have already destroyed user files. Pages reference blobs by id/hash only.
4. **InkSurface receives PRE-TRANSLATED drawing sessions.** Always `ds.Transform = mine * ds.Transform`. Never assign. Every rotation, mirror, layer transform and wrap-around tile below depends on this.
5. **It is a notes app first.** Anything that only serves a pro paint suite is marked niche or skip.
6. **Static ink cache (2500-stroke threshold) + uniform-grid spatial index** exist. Any feature that changes appearance without changing the element list (restyle, focus mode, blend modes, selection highlight) must bump the cache generation or it will silently render stale. This is the single most repeated trap in this document.

# PART 0 — Already in Quill, or already designed. Not offered as choices.

**Already built.** Vector element model (PenStroke / ShapeElement / TextElement, 14 PenTypes) · region/tile rendering on CanvasVirtualControl · ViewOffset/ViewZoom pan+zoom · static ink cache · uniform-grid spatial index · grids (None/Dotted/Square/Lines/Isometric/Triangle) + 1/2/3-point perspective overlay · page-size presets + artboard · eraser (object/point; hard-mask, soft-mask, slice, nudge) · pressure response curves (incl. 2-point simple) · per-preset stroke stabiliser · lasso selection with move/scale + Ctrl+D duplicate · shape recognition · tables · auto-grow text boxes · MathLive · comment pins · audio recording with playhead · PDF import · exports (vector PDF, SVG, HTML, per-section Markdown, PNG) · thumbnail cache · notebook gallery / recents / trash / tree · Ctrl+K command palette · AI assistant · voice dictation · keyboard presets + per-command rebinding · custom window caption · glow engine · LibraryStore (async, crash-safe) · SyncLog oplog collaboration · UndoRedo · CalcEngine.

**Already designed, not yet built.** These are approved; the items in Parts 1–4 that ride on them are marked `ext:planned`.
- **Layers** — LayerId on elements + a layer metadata list. Foundation for a large fraction of this catalogue.
- **Radial tool wheel** + top-bar removal + slim caption strip + single options menu.
- **.qpack** — versioned custom objects / stamps format; curated feed later, marketplace deferred.
- **Snapping engine** — key-point / grid / alignment snapping, Alt to suspend.
- **Art-in-notes vector paint** routed to a paint layer, and **oil paint** (raster tiles + impasto relighting) — spec'd, substrate under construction. Krita's colour-smudge engine is the same substrate and is folded into it.

Consequences worth stating once, because three researchers each rediscovered them:
- Fresco's "vector and pixel layers coexist with distinct icons, and the app refuses type-inappropriate tools rather than silently converting" is the correct model for Quill's layer kinds. Add `LayerKind { Vector, Raster, Adjustment }` from day one; gate the pen strip on the active layer kind; offer "create a raster layer above" instead of a silent conversion.
- Krita's advice to make the layer record a small polymorphic descriptor from day one (`{Id, Name, Kind, Opacity, BlendMode, Visible, Locked, LockAlpha, AlphaInherit, ParentGroupId, ClipToBelow, Transform, FamilyTag}`) is free now and expensive later. Every layer item in this catalogue assumes that record exists.
- Fresco's cloud-documents model is deliberately **not** copied; Quill's SyncLog-over-a-synced-folder gives the useful half without identity, quotas or a server bill. Version history (#20) is the one genuinely worth-porting piece and needs no cloud.

# PART 1 — Cross-app merged features

These capabilities appeared in two or three of the source apps. Each is one entry, with the apps

## 1.1 Construction aids, symmetry, snapping

### 1. Live symmetry: mirror axes + radial/mandala + multibrush — F, K
**What it does.** Turn on an axis and every stroke is mirrored live across it. Rotational mode repeats the stroke N times around a centre; combined with reflection that gives true mandala. Krita generalises further (Multibrush): symmetry, mirror, snowflake, translate copies, and manually-placed copies, up to 50.
**Why users love it.** Fresco: helpx `using/symmetry.html` documents reflective and rotational presets; the Adobe community feature-request thread (12730641) ran for years before it shipped. Krita: `reference_manual/tools/multibrush.html`; the mirror toggles live permanently in Krita's main toolbar, which tells you the usage rate. Pattern, lettering and diagram-border work.
**Quill how-to.** Purely a stroke-emission feature, so it composes with all 14 pen types and costs nothing at render time. Add `SymmetryDef { Kind: None|Vertical|Horizontal|Quad|Radial|Snowflake|Translate, CenterX, CenterY, Angle, Segments }` to the page — tens of bytes in library.json. In InkSurface's pointer pipeline, where a PenStroke is appended today, emit K sibling strokes whose point lists are transformed by precomputed `Matrix3x2` values (reflection about the axis, or `CreateRotation(2*pi*i/N)` about the centre, optionally each also mirrored for snowflake). Tag all K with a shared `MirrorGroupId` int so one undo removes the set and the eraser can optionally remove siblings. **Live preview before commit** costs nothing: draw the copies in the in-progress overlay pass by composing `Matrix3x2.CreateScale(-1,1,axis) * ds.Transform` around the same geometry — no geometry duplication until pointer-up. Draw the axis as a dashed line with a drag handle in the overlay pass. **Cap K at 24** for a notes app; Krita's 50 will thrash the uniform-grid index and blow the 2500-stroke cache threshold in a handful of gestures. Document the cap.
**Effort** M · **Depends on** none · **Status** new · **Verdict** strong-fit

### 3. On-canvas guides and assistants — F, K, C
**What it does.** Persistent geometric guides that live on the canvas and constrain freehand strokes near them: straight-edge ruler (Fresco), plus infinite and parallel variants (Krita), ellipse and concentric-ellipse, cubic-Bézier spline, arc, angle, rectangle, and polygon/circle stencils you can also outline or fill (Fresco). Concepts adds the interaction polish: handles resize, tap+drag moves, two-finger twist rotates, double-tap snaps the guide to its ideal form (arc→half-circle, angle→90°, ellipse→circle, rectangle→square), and numeric entry on tap+hold.
**Why users love it.** Fresco `using/ruler.html` — "the ruler automatically pins to that area and doesn't move around… rotation snapping is on by default"; the loudest Fresco-on-Windows complaint is a ruler limitation. Krita `user_manual/painting_with_assistants.html` — the parallel ruler is what technical and architectural users cite for consistent hatching. Concepts `manual/precision-tools` — MacStories names the Guide tool as what "enables uniform shape creation". This is the single most-requested construction feature across all three communities, and it is *different* from Quill's one-shot shape recognition: the guide persists, so you stroke along it many times with real brush texture.
**Quill how-to.** New `src/Quill/Models/AssistantDef.cs`: `{ GuideKind Kind, float[] Handles, uint Color, float Opacity, bool Enabled, bool Infinite }` held per NotePage next to PerspectiveDef — a handful of floats each, no library.json risk. Render in an overlay pass in `InkSurface.DrawRegion` **after** elements so guides sit on top of ink, with a cached `CanvasStrokeStyle` using `CustomDashStyle`; handles are 12 px circles hit-tested in **screen** space (`radius / ViewZoom`, or they become unusable when zoomed). Snapping lives in the input path — `AssistantSet.Snap(point, out snapped)` before the stabiliser:
- Line / parallel / infinite: project onto the segment, `t = dot(p-a, b-a)/|b-a|²`, clamped for finite, unclamped for infinite. Parallel projects onto the line through the **stroke's first point** with the ruler's direction.
- Ellipse: transform into ellipse space (`q = R⁻¹(p−C)` then divide by `(a,b)`), normalise to unit length, multiply and rotate back. One Newton refinement if you want true nearest point; five iterations is plenty at pen sample rates. **Concentric** variant skips the normalise and preserves `|q|`, capturing the ratio on PointerPressed and holding it for the whole stroke.
- Spline: flatten the cubic to ~256 segments into a cached `Vector2[]` on the def (invalidated by a handle-version counter); nearest point over the polyline with a coarse 16-segment early-out then refinement. Put this flattened-polyline nearest-point helper in a shared static `AssistantMath` — ellipse, spline and perspective all reuse it.
- Rectangle: nearest of four segments with corner clamping. Arc: project to the circle, clamp to the sweep.
- Blend raw→projected over a ~24 px falloff so entering and leaving a guide is not a hard jump.
**Stencil "mask" mode** (Fresco): wrap the stroke draw in `ds.CreateLayer(1f, geometry)` so paint clips inside or outside the stencil. Outline/Fill buttons emit a real ShapeElement so the result is a normal editable element. Reuse the planned snapping engine's Alt-to-suspend so the app has exactly one suspend modifier.
**Effort** L · **Depends on** snapping engine (`ext:planned`) · **Status** new · **Verdict** strong-fit

### 7. Snap controls: strength, snap-single, snap-eraser, align-vs-snap, traceback, allow-turns — K, C
**What it does.** Snapping as a set of distinguishable behaviours rather than one boolean. Krita: a 0–1000 **strength** slider so you get "guided but hand-drawn"; **Snap Single** commits a stroke to exactly one assistant for its whole length; **Snap Eraser** decides whether erasing snaps too; Ctrl+Shift+L toggles everything. Concepts: **Snap to Grid** pins the stroke to the lattice, **Align to Grid** only constrains *direction* so the stroke floats free but stays parallel to grid axes, **Allow Traceback** lets you retrace back along the line you just drew to undo part of it mid-stroke, **Allow Turns** lets a snapped stroke change direction at a grid node instead of forcing one straight run.
**Why users love it.** Krita `reference_manual/tools/freehand_brush.html` — Snap Single exists precisely because users complained about strokes flipping between overlapping assistants mid-line; inherit the fix, not the bug. Concepts `manual/precision-tools` — the align-vs-snap distinction is the subtle one people praise: clean angles without being trapped on the lattice.
**Quill how-to.** This is the concrete spec for the already-designed snapping engine. In InkSurface's pointer-move handler insert a filter stage **before** the stabiliser.
- Strength: `emit = Vector2.Lerp(raw, snapped, strength/1000f)`. Blend, never replace.
- Snap Single: resolve the winning assistant **once** on PointerPressed (nearest by perpendicular distance among enabled assistants inside a capture radius), store it in active-stroke state, query only that one for the rest of the stroke.
- Snap Eraser: one bool check against the current tool mode before calling Snap at all.
- Direction sets per grid: Square `{0°,90°}`; Isometric `{30°,90°,150°}`; Triangle `{0°,60°,120°}`; PerspectiveDef → for each vanishing point `v`, direction `normalize(p − v)`, recomputed per sample because it varies with position.
- Align mode: snap the *segment direction* to the nearest allowed direction within ~7°, then project each new sample onto the ray from the stroke's anchor. Snap mode: additionally quantise the anchor to the nearest grid intersection.
- Allow Turns: when the projected sample's distance from the anchor stops increasing **and** the raw direction is within tolerance of a *different* allowed direction, commit the run and re-anchor at the last projected point.
- Traceback: if the raw sample projects to a parameter `t` below the max `t` reached, truncate the point list back to `t`. Traceback and turns must share one state machine.
- Alt held = suspend all of it. Ctrl+Shift+L registered through the existing per-command rebinding system so it appears in the keyboard settings page and in Ctrl+K, never hardcoded.
Keep snap-while-drawing and snap-while-editing (#60) as genuinely independent toggles — collapsing them into one switch is the mistake Concepts deliberately avoids.
**Effort** M · **Depends on** snapping engine (`ext:planned`) · **Status** new · **Verdict** strong-fit

### 8. Perspective snapping with live preview ray, and shapes that map to the plane — F, K
**What it does.** Krita: strokes snap to rays converging on a vanishing point, and a dashed preview ray is drawn from the hovering cursor to the VP *before* you draw; 2-point uses two VPs plus a centre-of-vision and draws the correct horizon. Fresco: with a perspective grid active, drawn shapes and stencils automatically foreshorten onto the plane, and re-project as you drag them around rather than distorting.
**Why users love it.** Krita `painting_with_assistants` — "a preview line is drawn with all your snapped lines drawn to this line"; perspective assistants are the #1 reason non-painters download Krita. Fresco `using/grids-alignment.html` — "when you move the shapes around, they automatically map to the perspective plane"; architecture and interior sketchers rely on it.
**Quill how-to.** Quill already renders 1/2/3-point PerspectiveDef; the gap is snapping and preview. Fold PerspectiveDef into the AssistantDef model (keep back-compat deserialisation). Snap for `Kind=VanishingPoint`: on PointerPressed capture `d = normalize(firstPoint − VP)`, thereafter project each sample onto `VP + t*d`. For 2-point, pick whichever VP gives the smaller angular deviation from the initial drag direction and **stick with it for the whole stroke** (that is how Krita disambiguates). Live preview: hook the PointerMoved *hover* handler (pen hover is available on Surface hardware), draw a dashed ray at ~30% opacity, and guard invalidation to the union of the old and new ray bounds — do not repaint the page on every hover sample.
Shapes-on-plane: build the plane homography from the active PerspectiveDef (two VPs plus the ground line give the full 3×3 for each of the three planes). When a ShapeElement is created with "snap to plane" on, generate geometry in plane space and transform through the homography before storing. **Store the plane id and the plane-space parameters on the element** so a later drag re-projects rather than distorting an already-projected result — that is the difference between it feeling correct and feeling broken. Rendering is unchanged (stored points are screen-space); only creation and edit go through the homography. Add a floor / left-wall / right-wall selector to the existing perspective overlay chrome.
**Effort** L · **Depends on** PerspectiveDef (exists) + snapping engine · **Status** extends-existing · **Verdict** strong-fit

### 21. Grid customisation, draggable guides, paper backgrounds — K, C
**What it does.** Concepts: every grid type is individually configurable for spacing, line weight, colour, opacity and angle, plus ten canvas backgrounds (textured paper, blueprint, darkprint, transparent, custom). Krita: guides dragged out from rulers, snappable and lockable, with independent snap-to-grid / snap-to-guides toggles.
**Why users love it.** Concepts `manual/settings`; blueprint and darkprint presets show up constantly in shared architectural work. Krita's grids-and-guides docker — combined with assistants it is what makes Krita viable for layout and panelling.
**Quill how-to.** Quill's GridType enum exists but rendering constants are hard-coded. Add `GridStyle { Spacing, Weight, Color, Opacity, AngleDeg }` to NotePage and thread it through the grid pass. Angle: compose `Matrix3x2.CreateRotation(angle, pageCentre)` into the existing view transform (multiply, never assign). Backgrounds: textured paper is a `TurbulenceEffect` (fractal noise, low frequency) composited over a base colour with `BlendEffect` in Multiply, generated **once at load** into a tileable 512×512 CanvasRenderTarget and drawn as a tiled image — never regenerate per region. Blueprint/darkprint are just background-colour + grid-colour presets. Guides: `List<Guide{ bool Horizontal, float Position, bool Locked }>` on NotePage; show rulers along the artboard edge, drag from a ruler to create. Guides fold into the snapping engine as a candidate source alongside grid intersections and element key points. Ship **three** independent toggles — snap-to-grid, snap-to-guides, snap-to-elements — because Krita's single global snap toggle is what users end up fighting.
**Effort** M · **Depends on** snapping engine, existing grids · **Status** extends-existing · **Verdict** good-fit

## 1.2 Colour

### 5. Eyedropper: long-press loupe, samples the whole pen, layer-scope toggle — F, K, C
**What it does.** Press and hold anywhere and a magnifier loupe appears showing the pixel under the cursor with a colour ring; release to adopt it, no tool switch. Concepts goes further: the picker samples the *whole brush* — pen type, size, opacity, colour — so you can carry on drawing in exactly that tool, and shows current-vs-sampled side by side with an alpha toggle. Krita binds it to Ctrl+click.
**Why users love it.** Fresco `using/getting-started-with-user-interface.html` lists long-press-to-sample as a core gesture; community thread 10959427 and reviewer complaints that "the color picker picks from all layers" show how scrutinised it is — so ship the current-layer/all-layers toggle, which is the exact thing Fresco users complain is missing. Concepts `manual/colors` — brush-property extraction is what makes retroactive restyle (#51) round-trip: sample here, apply there.
**Quill how-to.** Take Concepts' vector-first version as primary and Fresco's loupe as the presentation. On pointer-hold past ~400 ms without movement, enter sample mode. **For a vector page, hit-test through the uniform-grid spatial index and take the topmost element's authored PenType / colour / width / opacity** — much better than raster sampling because you get the exact authored colour rather than an anti-aliased blend. Fall back to a raster read only over reference images, PDF imports and raster layers: render a small rect through the existing region-draw path into a pooled CanvasRenderTarget and `GetPixelColors` — never screen-grab the swap chain. Loupe: a Win2D-drawn circle in the overlay showing the same rect scaled up with `CanvasImageInterpolation.NearestNeighbor`, plus a ring filled with the centre colour and a split current/sampled swatch. Alpha toggle forces `A=255` when off. On release, set the pen and push to colour history (#14). Cost is one small offscreen render per sample.
**Effort** S · **Depends on** none · **Status** extends-existing · **Verdict** strong-fit

### 10. Colour wheel: harmony rules, shade strip, tonal / HSL / RGB surfaces — F, K, C
**What it does.** Fresco: a hue/saturation wheel with harmony rules (complementary, triad, analogous, split-complement) whose linked handles move together. Krita: a configurable selector (triangle-in-ring / square / wheel; HSV/HSL/HSI/HSY) paired with a **shade selector** strip offering lighter/darker/more-saturated variants of the current colour in one click. Concepts: three interchangeable surfaces in the wheel centre — a COPIC-style tonal ring organised by pigment and saturation with true black/white at the centre, an HSL wheel with three handles, and an RGB slider wheel, all numerically editable on tap+hold.
**Why users love it.** Adobe Color's harmony rules are the reference implementation people expect. Krita's shade strip is "the feature painters miss most in other apps" — picking a shadow colour becomes one click instead of a wheel hunt. Concepts' wheel is named in review after review ("the fantastic color wheel").
**Quill how-to.** **Do not build this from XAML gradient brushes** — WinUI 3 caches GradientStop mutations and a live-updating wheel will simply refuse to repaint. Two safe constructions, both used here:
- Continuous wheel: draw it once into a cached `CanvasRenderTarget` on a small `CanvasControl` (or compute per-pixel into a `byte[]` and `CanvasBitmap.CreateFromBytes` on hue change — 256×256 is sub-millisecond) and redraw only the handles per frame.
- Discrete tonal ring: generate 36 hue sectors × 5 saturation rings geometrically at startup in **OKLCH**, convert to sRGB in `Helpers/Util.cs`, plus an inner 11-step neutral ramp; render as WinUI `Path`/`ArcSegment` wedges. Discrete wedges dodge the gradient trap entirely. (Do not license COPIC; build the perceptual equivalent.)
Harmony is trivial HSV maths: given rule and base hue `h`, emit handles at `h+180`, `h±120`, `h±30`; dragging any handle rotates the set. Shade strip: 7–9 swatches walking value and saturation around the current colour **in HSY, keeping luma constant when varying hue** — that perceptual detail is what makes Krita's feel good. Persist last-used wheel type and last 12 colours in user settings, not per page. "Save as theme" writes swatches to the app palette file (#11).
**Effort** M · **Depends on** none · **Status** extends-existing · **Verdict** strong-fit

### 11. Palettes as real resources, plus auto-extraction from an image or page — K, C, F
**What it does.** Named palettes of swatches with groups, editable, storable globally or per-document, shippable inside resource packs (Krita). Concepts caps a palette at **eight** colours and puts them as a swipeable strip on the canvas under the wheel — deliberate ergonomics, palettes stay thumb-reachable — with drag-from-wheel-into-slot and shared read-only palettes. Fresco auto-generates a matching palette from any imported photo or placed asset.
**Why users love it.** Krita's resource-management docs and the trading of palettes on krita-artists; Concepts `manual/colors` for the 8-slot strip; Fresco `using/elements.html` — "Fresco automatically generates a color palette based on it, which helps you match your illustration style".
**Quill how-to.** `PaletteDef { Name, List<PaletteGroup{ Name, List<(Color, string)> }> }`. Two scopes: global palettes at `%LOCALAPPDATA%\Quill\palettes\*.json` (crash-safe temp+flush+atomic-replace), and one optional per-notebook palette on the Notebook record in library.json (~50 ARGB ints + names — negligible). On-canvas strip: an ItemsControl beside the wheel inside a `ScrollViewer` with `SnapPointsType.MandatorySingle` for swipe-between-palettes; drag-to-slot writes the colour at the pointer's slot index. Swatches render as Rectangles with `SolidColorBrush` — fine, because you replace the whole brush, which WinUI does honour. Import/export as `.qpal` (same JSON) so palettes travel through the SyncLog folder; reuse SyncLog's existing FileSystemWatcher rather than adding a second one. **Auto-extract**: downsample the source to 128×128 into a CanvasRenderTarget, `GetPixelBytes`, run median-cut to 5–8 buckets (or k-means k=6, 10 iterations — both ~60 lines), bias against near-greys unless the image is genuinely monochrome; sub-millisecond, run off the UI thread. For a vector page you do not need raster at all — elements already carry colours, so k-means the used colours directly ("Extract palette from page"). Surface palettes in Ctrl+K as `Palette: <name>`.
**Effort** S · **Depends on** none · **Status** extends-existing · **Verdict** good-fit

### 14. Colour history ring — K (F has plain recents)
**What it does.** An outer ring on the radial wheel holding the last ~10–30 colours actually used while painting, most-recent-first, one click to re-pick. A pure recency cache, separate from saved palettes.
**Why users love it.** Krita `reference_manual/popup-palette.html` — "the color history area shows the most recent color swatches that you have used while painting". Prominent enough that KDE bug 346534 was filed and fixed rather than the ring being dropped. Fresco's colour panel has the same idea as a flat swatch row.
**Quill how-to.** A ~24-entry deque of `Windows.UI.Color` as a static service beside ThumbnailCache; push on every committed stroke in the stroke-finalise path, de-duplicating by exact ARGB and moving an existing entry to the front. Persist to `%LOCALAPPDATA%\Quill\ui-state.json` (temp+flush+atomic-replace) — **never** library.json. Render as the outermost band of the wheel's CanvasControl: equal-angle wedges, `atan2` hit-test, each with a 1 px theme-border stroke so near-white swatches stay visible. Also expose "Recent colour N" in Ctrl+K so it works keyboard-only.
**Effort** S · **Depends on** radial tool wheel (`ext:planned`) · **Status** new · **Verdict** strong-fit

## 1.3 Selection and transform

### 4. Selection algebra and shape-select tools — F, K
**What it does.** Promote selection from a transient lasso to a real object: add / subtract / intersect / symmetric-difference with a shared Shift-adds / Alt-subtracts convention, rectangle / ellipse / polygon marquees, invert, feather the edge, and transform the *selection outline* independently of its contents.
**Why users love it.** Fresco `using/load-selection-to-edit-shapes.html` is literally titled "Transform your selections". Krita `user_manual/selections.html` — the shared modifier convention across all selection tools is repeatedly cited as what makes Krita learnable. This is the difference between a selection tool that is a toy and one people can finish work with, and it unlocks several other items here.
**Quill how-to.** Refactor the lasso result into `src/Quill/Models/Selection.cs` — a `SelectionRegion` holding a `CanvasGeometry` plus a boolean-op history. Every op is one built-in call: `CanvasGeometry.CombineWith(other, Matrix3x2.Identity, CanvasGeometryCombine.Union|Exclude|Intersect|Xor)`. Invert = Xor with the artboard rect. **Feather is a render-time property, not a geometry one**: rasterise the geometry into an alpha CanvasRenderTarget, `GaussianBlurEffect` at the feather radius, use that as the AlphaMask when applying any operation. Transform-selection-only = apply a Matrix3x2 to the geometry without touching elements. Hit-testing elements: fast AABB reject through the uniform-grid index, then `geometry.FillContainsPoint` on the stroke's sample points. Rect / ellipse / polygon tools are trivial drag handlers building a CanvasGeometry. Marching ants: `CanvasStrokeStyle` with `CustomDashStyle` and a `DashOffset` animated on `CompositionTarget.Rendering`. **Do this early** — #19, #45, #61, #73 all sit on it.
**Effort** M · **Depends on** existing lasso · **Status** extends-existing · **Verdict** strong-fit

### 19. Magic wand / select by colour with a live tolerance slider — F, K
**What it does.** Tap a region and everything contiguous and tonally similar is selected, with a Color Margin slider that widens or tightens the tolerance *after* the fact. Krita's variant adds select-similar-colour across the whole layer.
**Why users love it.** Fresco `using/magic-wand-tool.html`; shipped in v3.6 (May 2022) alongside Liquify and was one of the most-requested tools for years.
**Quill how-to.** New `Services/RegionSelect.cs` beside ThumbnailCache. Render the page (or active layer) once into a CanvasRenderTarget at view resolution, `GetPixelBytes` into a `byte[]`, scanline flood-fill from the tap point with tolerance = ColorMargin over Euclidean RGB distance, producing a bitmask. Trace the bitmask boundary with marching squares, simplify with Ramer-Douglas-Peucker, hand the path to the SelectionRegion from #4 so move / scale / duplicate work unchanged. **Re-running with a new margin only re-runs the flood fill on the cached `byte[]`, so the slider is interactive.** For a vector-only page offer a much cheaper second mode: hit-test through the uniform-grid index and select all elements whose colour is within tolerance of the hit element's. Ship both as "by area" and "by colour".
**Effort** L · **Depends on** lasso / SelectionRegion · **Status** extends-existing · **Verdict** strong-fit

### 17. Transform handles with skew, distort and perspective (homography) — F, K, C
**What it does.** Beyond move and uniform scale: drag corner handles to distort freely, two corners to skew, or map the bounding quad onto an arbitrary quadrilateral for a perspective correction. Concepts exposes it as one handle set with four behaviours depending on how many corners you activate — no mode switching. Krita additionally lets you drag the derived vanishing point directly.
**Why users love it.** Fresco `using/free-transform-tool.html` lists Scale / Skew / Distort / Perspective; Paperlike calls skew and distort what make Fresco viable for lettering and mockups. Concepts reviewers describe transforming as "fluid" precisely because there is no mode trip.
**Quill how-to.** Compute the 3×3 homography from four source corners to four dragged corners — the standard 8-unknown linear solve, ~40 lines with a small Gaussian elimination helper in `Helpers/Util.cs`. **Win2D's `ds.Transform` is `Matrix3x2` (affine only), so a homography cannot be pushed as a transform — you must transform the point arrays.** That is exactly what Quill's vector model makes easy and a raster app cannot do: apply `[x' y' w'] = H·[x y 1]`, divide by `w'`, per stroke point, on commit; re-fit each point's width by the square root of the local Jacobian determinant so the taper survives. Live preview at 60 fps: skew and distort are affine, so preview them with `ds.Transform`; for true projective preview use `Transform3DEffect` with a `Matrix4x4` built from the homography over a cached `CanvasCommandList` (Transform3D does support projective mapping; Transform2D does not). Rewrite geometry only on release. For the non-destructive case, store the 8 coefficients on the layer (see #46) and apply at draw time.
**Effort** L · **Depends on** lasso selection · **Status** extends-existing · **Verdict** good-fit

### 27. Cage and warp deformation — K, C
**What it does.** Warp deforms content via a grid or arbitrary anchor points using Rigid / Affine / Similitude algorithms; Cage draws a closed polygon around content and deforms it by dragging cage vertices. (Krita's mesh transform — a subdividable grid of Bézier patches for curved surfaces — is the same code plus one step.)
**Why users love it.** Krita's transform reference; Cage is the standard way artists bend an already-drawn arm or a hand-lettered banner without redrawing.
**Quill how-to.** Vector-native, in a new `src/Quill/Helpers/Deform.cs`. **Warp** = Moving Least Squares (Schaefer et al.): for each element point `p`, weights `w_i = 1/|p − src_i|^{2a}` over the anchors, then solve the small 2×2 least-squares system for the Similitude / Rigid / Affine variant and map `p` — ~30 lines of Vector2 maths, fine over a few thousand points. **Cage** = mean-value coordinates: precompute each point's barycentric weights against the cage polygon once when the cage closes, then any vertex drag is `sum(w_i * cageVertex_i)`. During drag, always transform a **cached copy of the original point arrays** — never accumulate, or you get drift. Commit on release through UndoRedo. Draw the cage/grid in the assistants overlay pass. Mesh transform is a small delta once Cage exists (per-cell bilinear `(u,v)` into a Coons/Bézier patch) — only worth adding if the paint layer gets real traction.
**Effort** L · **Depends on** SelectionRegion · **Status** new · **Verdict** good-fit

### 28. Liquify displacement brush — F, K
**What it does.** A brush that pushes content around like wet clay: push, twirl, pucker, bloat, offset, plus a Reconstruct/undo brush that paints the distortion back out locally. Each sub-brush has size, amount, flow, spacing and a reverse toggle.
**Why users love it.** Fresco `using/liquify-tool.html` (six modes, launched with Magic Wand in v3.6; character artists adopted it immediately for fixing proportions without redrawing). Krita's transform reference lists five brushes; Liquify is the most-demoed feature in Krita transform tutorials.
**Quill how-to.** Two routes; **take the vector one** — it is exact, cheap, and adds nothing to library.json.
- **Vector (recommended).** Maintain a sparse displacement field as a list of `{ centre, radius, Vector2 delta | rotation | scale }` deformers. When the liquify stroke ends, apply the accumulated field to the point arrays of every element whose AABB intersects the affected region (uniform-grid query). Per point, sum nearby deformers with falloff `w = smoothstep(1 − dist/radius)` and displace. Undo stores before/after point arrays for touched elements only.
- **Raster (only for the oil-paint substrate).** Per-layer `DisplacementRT` where R and G encode dx/dy around 0.5 grey — exactly what `DisplacementMapEffect` expects. Each dab draws a soft radial gradient of the right offset direction with `CanvasBlend.Add`: twirl is a tangential field, bloat/pucker radial, push aligned to drag delta. Render as `DisplacementMapEffect { Source = layerContent, Displacement = DisplacementRT, Amount = strength }`. Reconstruct draws neutral grey with alpha, decaying the field to identity. Undo is a tile delta. Mirror Fresco and restrict raster liquify to raster layers, and say so in the UI.
**Effort** L · **Depends on** none (vector route) · **Status** new · **Verdict** good-fit

## 1.5 Brushes

### 12. Brush editor: property × sensor curve matrix, spacing, scatter, taper, tilt — F, K
**What it does.** Every brush property (size, opacity, flow, rotation, scatter, roundness, softness) can be driven by any input sensor (pressure, tilt, twist, speed, fade, distance) through its own editable response curve; plus spacing, scatter on both axes, shape dynamics, taper at stroke start and end, and stylus tilt influence.
**Why users love it.** Fresco `using/pixel-brushes.html`; Ginny St. Lawrence's guide "Mastering Brush Taper Settings in Adobe Fresco" exists purely because taper is what makes a brush feel right. Krita `reference_manual/brushes/brush_engines.html` — the sensor/curve system across 17 engines is the number-one thing users cite when comparing Krita's brushes to competitors, and it is why brush packs feel distinct rather than being one brush at different sizes.
**Quill how-to.** Quill already has per-preset pressure curves including a 2-point simple mode — generalise the same curve widget into a `(Property, Sensor)` matrix: `Dictionary<BrushProp, SensorCurve>` on the pen preset, where `SensorCurve = { Sensor, control points, min, max }`. Sensors available from WinUI `PointerPoint.Properties`: Pressure, XTilt/YTilt (derive tilt magnitude and direction), Twist on supporting pens, plus computed Speed (px/ms between samples), Fade (samples since stroke start, with wrap) and Distance (accumulated arc length). Evaluate in the per-sample stamping loop: each property multiplies its base value by `curve.Eval(sensorValue)`. Add the flat parameters too: `{ Spacing, ScatterX, ScatterY, SizeJitter, AngleJitter, RoundnessJitter, AngleFollowsDirection, TaperInLen, TaperOutLen, TaperInWidth, TaperOutWidth, TiltInfluence }`. Taper is a width-multiplier ramp over the first/last N units of **arc length**. **Scatter and jitter must use a PRNG seeded from the stroke id** — a non-deterministic jitter shimmers on every pan and breaks tile redraw; this is the single trap in the feature. Build the editor as a scrollable list of properties with a curve thumbnail per row (Krita's exact layout) inside the existing PenPreset flyout, with a live preview stroke in a small CanvasControl re-rendering on every slider change. A few hundred bytes per preset; presets live in settings, not page data.
**Effort** L · **Depends on** existing pressure curves · **Status** extends-existing · **Verdict** strong-fit

### 22. Resource packs: manifest, licence, preview, tags, auto-tag on import — K, F, C
**What it does.** One compressed, versioned file holding many resources (presets, stamps, patterns, palettes, assistant sets, papers) with a manifest, importable in one click, with tags travelling inside the pack so an imported pack arrives already organised. ### 14b/29. Smudge and colour-pickup brushes — F, K
**What it does.** A brush that drags existing colour instead of depositing new colour. Krita distinguishes **Smearing** (literally smears underlying pixels) from **Dulling** (samples a disc of settable radius, preserves the brush shape, drops the picked colour over the trail length), with a Colour Rate that mixes in fresh paint.
**Why users love it.** Krita's colour-smudge engine is where most Krita painters live. Fresco's smudge is the light-editing kit users reach for after laying down flats.
**Quill how-to.** Raster-only; route into the oil-paint substrate. Per stamp: (1) sample the source disc by drawing the layer region into a small NxN CanvasRenderTarget through a `CropEffect`; (2) for **Dulling**, reduce to an average colour with a chain of downsampling `ScaleEffect` passes to 1×1 and read one pixel (cheap, GPU-side, no shader); (3) composite that colour, lerped with the brush colour by ColourRate, back at the destination through the brush's alpha mask with `AlphaMaskEffect` + `CompositeEffect` SourceOver. For **Smearing**, skip step 2 and blit the sampled tile offset by the stroke delta with the mask. Carrying the sample forward along the stroke is what produces the drag. All tiles under `%LOCALAPPDATA%\Quill\paint\<pageId>\`, atomically flushed. On a vector layer, offer the honest alternative — a "blend" pen that reduces stroke opacity and slightly displaces nearby endpoints — or require rasterising, as Fresco does.
**Effort** M–XL · **Depends on** oil-paint raster substrate (`ext:planned`) · **Status** new · **Verdict** niche

### 30. Stamp / texture brush engine and .abr import — F, C
See #14 in the pick-list ordering note; catalogued as **pick #14 group / see Part 2 (#38) and Part 4 (#69)**. Consolidated entry:
**What it does.** A brush whose tip is an image stamped along the stroke at a spacing interval, with rotation, scatter and jitter; Concepts adds a **Reveal** engine that unmasks an underlying grain so overlapping passes do not darken, and an in-app authoring UI with per-property variance graphs against pressure, tilt and velocity. Fresco's `.abr` import is how the entire third-party Photoshop brush economy (Kyle T. Webster, True Grit) reaches an app.
**Why users love it.** True Grit publishes an official "How to install Photoshop brushes in Adobe Fresco" guide; the Adobe thread about 21-of-25 brushes importing documents both that it works and where it breaks (legacy erodible tips). Custom brushes are the top request in Concepts' non-iOS community threads.
**Quill how-to.** Stamp rendering: along the tessellated stroke at `spacing * size` intervals, `DrawImage` the tip bitmap into a destination rect with a `ColorMatrixEffect` tinting it to the pen colour and rotation from the stroke tangent. **Seed all scatter/rotation randomness from the stroke GUID** — non-deterministic scatter breaks tile-based redraw. Reveal engine: render the stroke's coverage mask into a CanvasRenderTarget, then composite grain through `AlphaMaskEffect` so overlapping passes do not accumulate — the same built-in-composition pattern already proven with DistantSpecularEffect here. `.abr` v6/7/10 is a sampled-brush container: parse the 8BIM-style section headers, find the `samp` descriptor block, pull each tip's raw 8-bit greyscale bitmap plus spacing/diameter/angle/roundness, convert greyscale→alpha into PNGs under `%LOCALAPPDATA%\Quill\brushes\{packId}\`, register one pen preset per tip referencing the file. Skip legacy and computed-tip brushes with a clear count in the import summary — mirroring Fresco's own behaviour is acceptable and honest. Caveat: heavy stamping at high zoom stresses tile redraw — gate behind the ink cache and bake finished stamp strokes aggressively.
**Effort** L (.abr import) / XL (full authoring UI) · **Depends on** stamp brush engine, ink cache · **Status** new · **Verdict** good-fit (.abr) / niche (authoring UI)

### 6. Canvas rotation — K, C, F
**What it does.** Rotate the *view* (not the content) to any angle so the wrist can draw comfortable arcs. Krita puts a circular rotation slider on the popup palette plus key nudges and a reset; Concepts adds snapping at standard angles (disableable), a numeric readout you can tap to type an exact value, double-tap-to-recentre, and off-screen stroke pointers at the screen edges; Fresco binds two-finger rotate.

**Quill how-to.** Add `float ViewRotation` next to ViewOffset/ViewZoom. Grid and guide rendering already run in canvas space, so they rotate for free. **Cache in document space and rotate at composite time so rotation stays free**, rather than invalidating the 2500-stroke static cache on every rotation change. Two-finger rotate extends the existing pinch handling with the angle delta; snap to a multiple of 15° when within 4° on gesture end. Readouts become editable TextBoxes in the planned slim caption strip. Off-screen indicators: draw a small chevron at the viewport edge toward each off-screen element's centre, capped at the 8 nearest.

### 20. Page version history and named snapshots — F, K
**What it does.** Fresco: every cloud document keeps prior versions with a 30-day recovery window and one-click rollback. Krita: named in-memory **snapshots** (shallow copies) you can jump back to — a branching alternative to linear undo, used to try a colour scheme without fear.

### 24. Timelapse recording and export — F, K
**What it does.** The app records the session and can export a sped-up video of the artwork building itself, with quality tiers. Krita's Recorder docker writes a numbered image sequence at an interval while you work.

### 36. Perspective grid capture from an image

### 38. Live watercolour brush (wet-in-wet bloom, granulation, edge darkening)
**What it does.** A brush that deposits pigment into a persistent wet film. Paint keeps spreading after the stroke ends, blooms into neighbouring wet areas, mixes where films overlap, and settles into paper texture as it dries.
**Why users love it.** helpx `using/live-brushes.html` calls live brushes the headline feature; Creative Bloq's review calls them "the slickest implementation of digital watercolours, with liquids that blend and bleed right in front of your eyes"; deke.com and the Paperlike 2025 deep-dive both single it out as the one thing Fresco does that no competitor matches.
**Quill how-to (no shader anywhere — every step is a Win2D built-in).** A `WatercolorLayer` raster substrate reusing the oil-paint raster-tile machinery. Per dirty tile keep two `CanvasRenderTarget`s: **PigmentRT** (RGBA, colour + concentration in alpha) and **WetnessRT** (single channel packed into alpha). A stroke stamps pigment with `CanvasBlend.Add` and stamps wetness simultaneously. Diffusion runs on a `CompositionTarget.Rendering` tick capped at ~15 Hz, only over the dirty tile set from the uniform-grid index; each iteration is (a) `GaussianBlurEffect` radius 1.5 on PigmentRT, (b) `DisplacementMapEffect` driven by a cached `TurbulenceEffect` tile (fractal sum, ~0.05 frequency) for ragged-edge irregularity, (c) `AlphaMaskEffect` against WetnessRT so pigment only spreads where the film is still wet, (d) composite back with `CompositeEffect` SourceOver. **Edge darkening (the classic watercolour ring) falls out free**: subtract the blurred field from the original with `ArithmeticCompositeEffect` and add the positive residue back at the boundary. Drying = multiply WetnessRT by ~0.985 per tick via `ColorMatrixEffect` on the alpha row; when max wetness over a tile drops below epsilon, stop ticking that tile and bake it into the flattened raster. Granulation = multiply the deposit by a tiled paper-grain texture (pre-rendered Turbulence cached once as a CanvasBitmap). Persist only the flattened PNG per layer at `%LOCALAPPDATA%\Quill\raster\{pageId}\{layerId}.png`, temp+flush+atomic-replace; library.json gets the layer id, blend mode and a file hash and nothing else.
**Effort** XL · **Depends on** layers + raster-tile substrate (`ext:planned`) · **Status** new · **Verdict** good-fit

### 40. True stabilizer: dead-zone delay, speed-adaptive sample count, finish line
**What it does.** Not a moving average. The cursor drags a circle of radius `delay`; the brush only moves when the cursor leaves that circle, and the pulled point is the average of the last *sample count* positions — so long strokes are perfectly smooth while sharp corners are still possible. **Finish Line** animates the lag out to the true release point instead of truncating the stroke. **Stabilize Sensors** low-passes pressure and tilt through the same buffer.
**Why users love it.** `reference_manual/tools/freehand_brush.html`; 5.3 added Sample Count at Max/Min Speed, Delay ("creates a dead zone for sharper corner control"), Finish Line and Stabilize Sensors. It is the single most-recommended setting in every "how do I get smooth lines in Krita" thread. **For Quill this is not a painting feature — it is handwriting legibility, which is the product.**
**Quill how-to.** Replace/augment the existing per-preset stabiliser algorithm in the pointer-move accumulation. Keep a ring buffer of the last N raw samples. Each move: if `distance(raw, lastEmitted) < delayRadius`, emit nothing (dead zone); otherwise pull the emitted point toward the buffer mean by a factor derived from sample count. Make sample count **speed-adaptive** — lerp between `SampleCountMinSpeed` and `SampleCountMaxSpeed` on instantaneous cursor speed so fast strokes stay responsive. On PointerReleased, if Finish Line is on, run a short `CompositionTarget.Rendering` tick emitting the remaining buffered points over ~80 ms so the stroke reaches the true release position. Stabilize Sensors is a separate flag that low-passes pressure/tilt through the same buffer **before** they hit the pressure response curve. Surface delay / sample-count / finish-line in the PenPreset flyout next to the existing slider; three extra floats per preset.
**Effort** M · **Depends on** existing stabiliser · **Status** extends-existing · **Verdict** strong-fit

### 53. Nudge tool — push a line like a string
**What it does.** A tool that locally deforms a stroke: drag near a line to push it away with a falloff, or grab and pull it directly — fixing a curve without redrawing or entering a point-edit mode.
**Why users love it.** `manual/brushes-and-tools` — "an intuitive way to edit strokes — push lines like string or pull directly". It is also one of the assignable Finger Actions, meaning TopHatch considers it a constant-use tool, not an occasional one.
**Quill how-to.** Quill already has a "nudge" **eraser style** — promote it to a first-class `ToolType.Nudge` with its own radius and strength. On pointer-down, query the uniform-grid index for strokes intersecting the brush circle. Per move delta `d`, for each point within radius `r`: `p += d * w` where `w = smoothstep(1 − dist(p,cursor)/r) * strength`. **Push mode uses only the component of `d` normal to the local tangent so the stroke slides rather than bunching**; pull applies `d` directly. After moving, re-run the arc-length resampler if any adjacent gap exceeds 2× the original spacing, or long drags create visible polygonal kinks. Invalidate only the swept-rect union per move; bump the ink-cache generation **once on pointer-up**, not per move. Undo captures affected point arrays once on pointer-down.

### 58. Retroactive smoothing — re-smooth an existing stroke on a 0–100 slider
**What it does.** Smoothing as a first-class stroke *property*, not a one-time input filter: 0% is raw hand input, 50% removes bumps, 100% collapses the stroke into a perfectly straight line — dialled after the fact on a selection.

**Why users love it.** `manual/brushes-and-tools` — "Wire Tool: line width stays uniform at any zoom level"; the export tutorial recommends Wire or Fixed Width for DXF-bound linework.
**Quill how-to.** Add `Wire` to the PenType enum. In the stroke renderer, branch: instead of tessellating a variable-width ribbon, build a `CanvasPathBuilder` polyline and call `ds.DrawGeometry(geo, colour, strokeWidth / ViewZoom, cachedStrokeStyle)` so the device-space width is constant. **Two consequences to handle deliberately: (1) hit-testing must use the same zoom-scaled tolerance; (2) wire strokes must be excluded from the static ink cache**, because a bitmap baked at one zoom scales wrongly — tag the PenType in the cache-eligibility predicate. Exporters emit these at a fixed 0.25 pt / 0.25 px hairline.
**Effort** M · **Depends on** none · **Status** new · **Verdict** good-fit

### 75. DXF (R14) export with wireframe mode
**What it does.** Export as an AutoCAD DXF R14 file with vector paths and basic colour, for CAD or a laser/waterjet cutter, with a wireframe option that strips brush character down to centrelines.
**Why users love it.** `manual/export` — "DXF: CAD format (R14) by Autodesk for technical drawings and laser cutting"; a real production path (artboard sizes matched to cutting services).
**Quill how-to.** `Services/DxfExporter.cs` sharing PdfExporter's element enumeration and unit conversion. **DXF R14 ASCII is trivially writable by hand — no library.** Emit SECTION/HEADER with `$INSUNITS` (4 = mm, 1 = inch) from DrawingScale; SECTION/TABLES with one LAYER per Quill layer; SECTION/ENTITIES with PenStroke → `LWPOLYLINE` (group codes 0 / AcDbEntity / 8 layer / 90 vertex count / 70 closed flag / 10,20 per vertex), rect → closed LWPOLYLINE, ellipse → `ELLIPSE` (10,20 centre; 11,21 relative major-axis endpoint; 40 ratio), TextElement → `TEXT`. Colour via ACI index 62 with a static nearest-match table. Wireframe drops per-point widths and emits centrelines; non-wireframe emits the tessellated ribbon outline. **Multiply canvas DIP by DrawingScale so 1 DXF unit = 1 real unit — unit mismatch is the #1 reported failure mode, so write `$INSUNITS` and also state the units in the export dialog.** Bound the export to the artboard rect. Pairs with #59 (closed contours) and #74.
**Effort** L · **Depends on** drawing scale (#54) · **Status** new · **Verdict** good-fit

### 2. Reference images and underlays — F, K, C (merged; catalogued here because Concepts' version is the strongest)
**What it does.** External images placed on the canvas outside the layer stack: movable, scalable, rotatable, per-image opacity and saturation, colour-pickable directly, and persisted with the page. Fresco's flavour is a small floating always-on-top panel you can zoom independently of the canvas. Krita's is an on-canvas reference tool/docker with session-only or embedded options. Concepts' is an **underlay**: images and PDFs import *to scale*, land selected for positioning, auto-create a bottom layer, and default to locked so you trace over them.
**Why users love it.** Krita `reference_images_docker` — "pick an image from outside of Krita and use it as a reference, and you can pick colors from it directly"; the community wanted it so badly a third-party plugin existed before the native tool shipped. Concepts `manual/import` — tracing over a plan or screenshot is the dominant Concepts workflow in architecture reviews, and scale calibration (#56) is built directly on it. Fresco's floating reference is the #1 Procreate-parity request in the Adobe forums. **For Quill this is squarely a notes feature: pin a lecture slide, a photographed whiteboard, or a textbook figure beside your notes.**
**Quill how-to.** New `Services/ReferenceImageService.cs` beside ThumbnailCache. References are **not elements**: keep a per-page list of `{ BlobId, Matrix3x2 Placement, Opacity, Saturation, Locked, KeepExternal }`. **Copy the source into a content-addressed store `%LOCALAPPDATA%\Quill\blobs\{sha256}.png` (re-importing the same reference then costs nothing) with temp+flush+atomic-replace, and store only the id in library.json — this is the whole ballgame for not bloating the 53 MB file.** Decode via `CanvasBitmap.LoadAsync` into a byte-capped LRU keyed by BlobId, mirroring ThumbnailCache's eviction, evicting on page switch. "To scale": if the file carries DPI metadata, size the destination so physical size is honoured against DrawingScale (#54); otherwise fit to 50% of the viewport. Draw **first** in DrawRegion (before grid and strokes) with `ds.DrawImage(bitmap, dest, source, opacity)` wrapped in a `SaturationEffect`. Default `Locked = true` with an opacity slider in the selection bar — the two things every tracing workflow immediately reaches for. Manipulation handles reuse the lasso scale/rotate handles. Colour-pick reads from a cached `GetPixelColors` of the region rather than per-pick round trips. **Blob GC: on save, sweep blobs unreferenced by any page, with a grace period so undo still works.** Ship "open reference" in Ctrl+K. If a floating always-on-top panel is also wanted (Fresco's flavour), it is a Grid overlay hosting a small CanvasControl with a ManipulationDelta-driven TranslateTransform, a corner resize thumb and a collapse chevron — same storage, different presentation.
**Effort** M · **Depends on** none (layers optional) · **Status** extends-existing (PDF import exists) · **Verdict** strong-fit

# PART 5 — Deliberately skipped, with reasons

| Feature | App | Why skip |
|---|---|---|
| **Cloud documents as the primary store** | F | Quill's SyncLog-over-a-synced-folder is deliberately provider-agnostic and already delivers multi-device convergence without identity, billing, quotas, an offline story or a server bill. Fresco's own community complains about being unable to work purely locally. The one worth-porting piece is version history (#20), which needs no cloud. |
| **Live watercolour/oil behaviour on a vector layer** | F | A fluid simulation has no vector representation; "vector watercolour" can only mean pre-baked texture fills that would look wrong across Quill's zoom range and still need per-stroke raster storage. Give the honest answer Fresco gives — live media go on raster layers — and make the layer kind obvious *before* the user commits a stroke (raster badge in the panel, a one-time "this creates a raster layer" prompt). Recording the authoring resolution of a raster layer avoids the pixelation complaints Fresco gets. |
| **Full animation timeline with tweening, curves docker, audio sync, storyboards** | K | Either N element lists per page (multiplying an already-53 MB library) or a whole second document model, plus a timeline UI that would dominate the chrome Quill is actively removing. The useful 10% is #26 (ghost adjacent pages); the audio-with-playhead need is already served. If motion is ever wanted, do it as an export-time GIF from a page range, not a document-model change. |
| **Pexels stock-image search in-app** | C | Requires a third-party API key and network egress in an app whose only current network dependency is an optional AI assistant — a privacy-surface expansion for a notes app. Every fetched image is a raster blob to store and GC, poor value per megabyte next to the existing library problem. And it serves mood-boarding, not pen-first note-taking. The cheaper win — system file picker plus paste-from-clipboard — is already covered by #2. |

## Build-order notes (editor's view)

- **Do #4 (selection algebra) early** — #19, #45, #61, #73 and much of the transform work sit on it.
- **Do #54 (drawing scale) before #55, #56, #57, #75** — they all read one record.
- **Do #43 (LOD) before any feature that adds draw passes** — blend modes, focus mode, onion skin, masks and stencils all add composition cost, and the 53 MB library is already the stress case.
- **#3 (assistants) and #7 (snap controls) are one project** in practice; #8, #47, #59, #60 extend the same subsystem.
- **The ink-cache generation counter is the recurring bug source.** #51, #62, #67, #9, #46 and #74 all change appearance without changing the element list. Add an explicit `CacheGeneration` bump API before building any of them, and a debug overlay that shows the current generation.
- **Content-addressed blob store** (`%LOCALAPPDATA%\Quill\blobs\{sha256}`) is needed by #2, #30, #38, #64, #69 — build it once, with the unreferenced-blob sweep and grace period, rather than five times.
